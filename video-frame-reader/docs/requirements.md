# video-frame-reader 要件定義書

## 背景と目的

Claude Codeで動画を直接渡すと、5秒のMP4を全フレーム分割するだけで約450円のコストがかかる。差分の小さいフレームを間引いてキーフレームのみを1枚の画像にまとめることで、約24円（約95%削減）に抑えられる。

このスキルは「言葉で説明しにくいアニメーションや画面遷移のバグ」をClaudeに正確に伝えるために設計された。

## ユースケース

1. **アニメーションバグの報告**: 想定と異なるUI遷移・タイミングのずれをスクショでは伝えにくい
2. **歌詞ハイライトなど複雑なUI**: 複数の状態が時系列で変化する動作の確認
3. **スクロールアニメーション**: 自動スクロール動作の検証

## 機能要件

| # | 要件 | 優先度 |
|---|------|--------|
| F1 | MP4/GIF/MOV の読み込み | 必須 |
| F2 | ピクセル差分によるキーフレーム抽出 | 必須 |
| F3 | 横並びストリップへの合成 | 必須 |
| F4 | タイムスタンプ付きキャプション | 必須 |
| F5 | JPEG/PNG 出力の切り替え | 必須 |
| F6 | 差分しきい値のパラメータ化 | 必須 |
| F7 | コスト比較表の出力 | 必須 |
| F8 | パス省略時のプロジェクト内自動検索 | 必須 |
| F9 | /vfr エイリアス | 必須 |

## 非機能要件

- 5秒/30fps の動画を10秒以内に処理できること
- 出力ファイルは `/tmp/` に書き出し、会話終了後は不要
- Python 3.12 / opencv-python / Pillow + uv で動作

## パラメータ仕様

| パラメータ | 型 | デフォルト | 範囲 | 説明 |
|-----------|-----|-----------|------|------|
| `video_path` | str | 省略可 | — | 入力動画パス。省略時は自動検索 |
| `--format` | str | `jpeg` | jpeg/png | 出力画像形式 |
| `--threshold` | float | `30.0` | 0〜100 | キーフレームしきい値 |

## コスト試算（参考）

Claude Sonnet 4.x 入力トークン $3/MTok、1 USD = 150 円として試算。

| 方法 | 推定トークン | 推定コスト |
|------|------------|------------|
| 全フレーム PNG（5秒/30fps/720p） | ~184,000 | ~250円 |
| 全フレーム JPEG（同上） | ~184,000 | ~83円 |
| キーフレームのみ JPEG（典型例） | ~1,200 | ~0.5円 |

※ 実際のコストはビデオ内容・解像度・抽出フレーム数により変動する。
